## 1. Core Principles of Technical Guidance

### 1.1 Effective Communication
```markdown
## Clear Communication
- Use simple, non-technical language
- Avoid jargon and acronyms
- Confirm understanding
- Provide examples
- Use analogies when helpful

## Active Listening
- Let users explain fully
- Ask clarifying questions
- Paraphrase to confirm
- Note important details
- Show empathy and patience
```

### 1.2 User-Centric Approach
```markdown
## Understanding User Needs
- Assess technical proficiency
- Identify business impact
- Consider user environment
- Respect time constraints
- Adapt to learning style

## Building Rapport
- Be approachable and friendly
- Use positive language
- Show genuine interest
- Be patient and encouraging
- Follow up consistently
```

## 3. Technical Guidance Techniques

### 3.1 Problem-Solving Framework
```markdown
## The 5-Step Method
1. **Gather Information**
   - Ask open-ended questions
   - Document symptoms
   - Note error messages
   - Check system details

2. **Diagnose the Issue**
   - Replicate if possible
   - Check logs and events
   - Identify patterns
   - Determine root cause

3. **Develop a Solution**
   - Research if needed
   - Consider alternatives
   - Assess risks
   - Prepare rollback plan

4. **Implement the Fix**
   - Explain steps clearly
   - Get user confirmation
   - Make one change at a time
   - Document each step

5. **Verify and Close**
   - Test the solution
   - Confirm with user
   - Document resolution
   - Schedule follow-up
```

### 3.2 Teaching Technical Concepts
```markdown
## Explanation Techniques
- Start with the "why"
- Use analogies
- Break into smaller parts
- Provide real-world examples
- Check for understanding

## Demonstration Tips
- Go slowly
- Explain each step
- Encourage questions
- Have them try it
- Provide practice exercises
```
