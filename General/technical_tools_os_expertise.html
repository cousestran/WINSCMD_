4. Troubleshooting Methodologies

4.1 Structured Approach

1. **Information Gathering**
   - Identify symptoms and error messages
   - Determine scope and impact
   - Check system logs and event viewers

2. **Reproduction**
   - Replicate the issue
   - Document steps to reproduce
   - Identify patterns

3. **Diagnosis**
   - Check common causes first
   - Use diagnostic tools
   - Review recent changes

4. **Resolution**
   - Apply known solutions
   - Test potential fixes
   - Document the solution

5. **Verification**
   - Confirm resolution with user
   - Ensure no new issues introduced
   - Update documentation

6. Interview Preparation

6.1 Common Technical Questions

## Windows-Specific
"How would you troubleshoot a Windows 10 machine that's running slowly?"
1. Check Task Manager for high CPU/memory usage
2. Review startup programs (Task Manager > Startup)
3. Check disk space and run disk cleanup
4. Run SFC and DISM scans
5. Check for Windows updates
6. Review event logs for errors
7. Check for malware

## macOS-Specific
"A user reports their Mac is running slowly. How would you diagnose this?"
1. Check Activity Monitor for resource usage
2. Review login items (Users & Groups > Login Items)
3. Check storage (About This Mac > Storage)
4. Run First Aid in Disk Utility
5. Check for macOS updates
6. Review Console logs
7. Check for kernel_task issues

6.2 Scenario-Based Questions

## Remote Support Scenario
"A remote user can't connect to the VPN. Walk me through your troubleshooting steps."
1. Verify internet connectivity
2. Check VPN credentials and MFA
3. Verify VPN client installation/version
4. Check firewall/antivirus settings
5. Test with different network
6. Check VPN server status
7. Review VPN logs
8. Escalate if needed

## Cross-Platform Issue
"A user reports they can't print from their Mac to a network printer that works for Windows users."
1. Verify printer is shared correctly
2. Check printer drivers on Mac
3. Verify network connectivity
4. Check printer queue on Mac
5. Test with different document
6. Review system.log for print errors
7. Try adding printer with IP address
